## Research: Server-Sent Events (SSE) with Hono and TanStack Query

### Answer

Hono has **native, first-class SSE support** via the `streamSSE()` helper (from `hono/streaming`), which handles streaming responses with proper content-type headers and connection management. On the frontend, TanStack Query doesn't have built-in SSE integration — you must use `EventSource` directly with `useEffect`, then update cache via `queryClient.setQueryData()` for direct updates or `queryClient.invalidateQueries()` to trigger refetches. The best pattern separates **cache invalidation signals** (sent via SSE) from **data payloads** (fetched on-demand via existing REST endpoints).

### Key Findings

1. **Hono has built-in SSE support via `streamSSE()` helper**
   - Imported from `hono/streaming`
   - Handles streaming response setup, chunked encoding, and headers automatically
   - Supports `event` type, `id`, and `data` fields per event
   - Includes `stream.sleep()` for delays and `stream.onAbort()` for cleanup
   - Works across Cloudflare Workers, Node.js, Deno, and Bun (source: https://hono.dev/docs/helpers/streaming)

2. **TanStack Query does NOT have native SSE support**
   - No built-in `useQuery` option for streaming
   - Pattern: Manual `EventSource` inside `useEffect`, then update cache
   - Two approaches:
     - **Direct update**: `queryClient.setQueryData(queryKey, newData)` — immediate UI update
     - **Invalidation**: `queryClient.invalidateQueries({ queryKey })` — triggers refetch for subscribed queries
   - (sources: https://stackoverflow.com/questions/75901223/react-query-and-sse, https://rustedcompiler.medium.com/using-eventsource-sse-with-react-query-b72e20923d8c)

3. **Cache invalidation via SSE is the production pattern (RTK Query proven approach)**
   - Instead of streaming full data, stream only **cache invalidation signals** with tag names
   - Client dispatches `queryClient.invalidateQueries({ queryKey })` on event
   - Only actively subscribed queries automatically refetch
   - Reduces SSE payload size and network overhead
   - TanStack Query supports tag-based invalidation via `meta` field (not built-in like RTK Query, but achievable)
   - Proven in production by Dopamine (NestJS + Next.js + React using RTK Query pattern)
   - (source: https://www.dopamine.dev/blog/sse-rtk-query-cache-invalidation)

4. **Event design for experiments: aggregate updates > individual cell completions**
   - Raw streaming of every cell completion is noisy and creates high cardinality
   - **Recommended pattern**: Emit aggregate `status_updated` or `progress_updated` events per experiment run
   - Include: `experimentId`, `status` (running/completed/failed), `timestamp`, optional aggregated metrics (cells completed, success rate)
   - Avoid per-cell streaming unless UI explicitly shows individual cell state
   - Event aggregation reduces noise and simplifies cache invalidation logic
   - (source: https://oneuptime.com/blog/post/2026-01-30-event-aggregation/view and event-driven architecture patterns)

5. **Hono streaming helpers include error handling**
   - Optional third-argument error handler: `stream(c, async (stream) => {...}, (err, stream) => {...})`
   - `stream.onAbort()` callback for cleanup when client disconnects
   - Errors during streaming cannot overwrite response (headers already sent), so handler must write to stream
   - (source: https://hono.dev/docs/helpers/streaming)

### Recommended Approach

#### **Hono Backend (Server)**

```typescript
import { Hono } from 'hono'
import { streamSSE } from 'hono/streaming'

const app = new Hono()

// Experiment status SSE endpoint
app.get('/experiments/:id/events', async (c) => {
  const experimentId = c.req.param('id')
  
  return streamSSE(c, async (stream) => {
    // Optional: send initial connection event
    await stream.writeSSE({
      event: 'connected',
      data: JSON.stringify({ experimentId, timestamp: new Date().toISOString() }),
      id: '0'
    })

    // Subscribe to experiment status updates (from pub/sub, event emitter, or WebSocket bridge)
    const unsubscribe = experimentService.on(`experiment:${experimentId}:update`, (update) => {
      // Example update: { status: 'running', cellsCompleted: 42, totalCells: 100, elapsedMs: 5000 }
      stream.writeSSE({
        event: 'status_updated', // ← not raw data, just invalidation signal
        data: JSON.stringify({
          experimentId,
          status: update.status,
          cellsCompleted: update.cellsCompleted,
          totalCells: update.totalCells,
          timestamp: new Date().toISOString()
        }),
        id: String(update.id)
      })
    })

    // Handle client disconnect
    stream.onAbort(() => {
      unsubscribe()
      console.log(`Client disconnected from experiment ${experimentId}`)
    })

    // Keep connection open
    while (true) {
      await stream.sleep(30000) // Ping every 30s to keep connection alive
    }
  }, (err, stream) => {
    // Error handler
    stream.writeln(`data: {"error": "Stream error"}`)
    console.error(`SSE error for experiment ${experimentId}:`, err)
  })
})

export default app
```

#### **React + TanStack Query Frontend (Client)**

**Step 1: Define query options (no SSE needed here)**
```typescript
import { queryOptions } from '@tanstack/react-query'

export const experimentQueryOptions = (experimentId: string) =>
  queryOptions({
    queryKey: ['experiments', experimentId],
    queryFn: async () => {
      const res = await fetch(`/api/experiments/${experimentId}`)
      return res.json()
    },
    staleTime: 1000 * 60 * 5 // 5 minutes
  })
```

**Step 2: Custom hook for SSE cache invalidation**
```typescript
import { useEffect } from 'react'
import { useQueryClient } from '@tanstack/react-query'

export function useExperimentSSE(experimentId: string) {
  const queryClient = useQueryClient()

  useEffect(() => {
    const eventSource = new EventSource(`/api/experiments/${experimentId}/events`, {
      withCredentials: true
    })

    eventSource.addEventListener('connected', (event) => {
      console.log('SSE connected:', event.data)
    })

    // Listen for status updates and invalidate cache
    eventSource.addEventListener('status_updated', (event) => {
      const update = JSON.parse(event.data)
      console.log('Experiment status update:', update)

      // Approach A: Direct update (if you want instant UI feedback)
      queryClient.setQueryData(['experiments', experimentId], (old) => ({
        ...old,
        status: update.status,
        cellsCompleted: update.cellsCompleted,
        totalCells: update.totalCells
      }))

      // Approach B: Invalidate to trigger refetch (if you want fresh data from server)
      // queryClient.invalidateQueries({ queryKey: ['experiments', experimentId] })
    })

    eventSource.onerror = () => {
      console.error('SSE connection error')
      eventSource.close()
    }

    return () => {
      eventSource.close()
    }
  }, [experimentId, queryClient])
}
```

**Step 3: Use in component**
```typescript
import { useSuspenseQuery } from '@tanstack/react-query'
import { experimentQueryOptions } from './queries'
import { useExperimentSSE } from './hooks'

export function ExperimentDetail({ experimentId }: { experimentId: string }) {
  // Establish SSE connection
  useExperimentSSE(experimentId)

  // Standard TanStack Query hook
  const { data: experiment, isLoading, error } = useSuspenseQuery(
    experimentQueryOptions(experimentId)
  )

  if (isLoading) return <div>Loading...</div>
  if (error) return <div>Error: {error.message}</div>

  return (
    <div>
      <h2>{experiment.name}</h2>
      <p>Status: {experiment.status}</p>
      <progress value={experiment.cellsCompleted} max={experiment.totalCells} />
      <span>{experiment.cellsCompleted}/{experiment.totalCells} cells</span>
    </div>
  )
}
```

### Design Decision: Event Types

**✅ Recommended: Aggregate Status Events**
```typescript
// Emit periodically (every 2-5 seconds) or on major state changes
streamSSE({
  event: 'status_updated',
  data: JSON.stringify({
    experimentId,
    status: 'running',           // running | paused | completed | failed
    cellsCompleted: 42,          // aggregate count
    totalCells: 100,
    successRate: 0.95,           // aggregate metric
    elapsedMs: 5000,
    estimatedRemainingMs: 12000,
    timestamp: '2026-03-12T23:01:33Z'
  })
})
```

**❌ Avoid: Per-Cell Streaming**
```typescript
// DON'T do this — creates high cardinality and noise
streamSSE({
  event: 'cell_completed',
  data: JSON.stringify({ cellId: 'abc-123', status: 'success', durationMs: 150 })
})
// x100 cells = 100 separate SSE events per run
```

**Optional: Two-Event Pattern (Advanced)**
If you need both real-time UI updates AND server-side freshness:
1. **`status_updated`** (via SSE): Optimistic UI update with aggregate metrics
2. **`refetch_required`** (via SSE): Triggers `queryClient.invalidateQueries()` to fetch latest from server

```typescript
eventSource.addEventListener('refetch_required', () => {
  queryClient.invalidateQueries({ queryKey: ['experiments', experimentId] })
})
```
