## Research: Vercel AI SDK + OpenRouter Configuration

### Answer

Use the **`@openrouter/ai-sdk-provider`** package (maintained by OpenRouter) to integrate OpenRouter with Vercel AI SDK. Initialize the provider with your `OPENROUTER_API_KEY`, then call `createOpenRouter()` to create a provider instance. For structured output, use `generateObject()` with a Zod schema—it works seamlessly with OpenRouter models without any additional baseURL configuration.

### Key Findings

1. **Official Provider Package** — OpenRouter provides a dedicated `@openrouter/ai-sdk-provider` npm package (570 GitHub stars) maintained by the OpenRouter team. This is the **recommended approach**, not a generic OpenAI-compatible setup (source: [OpenRouter AI SDK docs](https://openrouter.ai/docs/guides/community/vercel-ai-sdk), [GitHub repo](https://github.com/OpenRouterTeam/ai-sdk-provider))

2. **Installation & Setup** — Install with `npm install @openrouter/ai-sdk-provider` and import `createOpenRouter`. Initialize with `createOpenRouter({ apiKey: process.env.OPENROUTER_API_KEY })`. No custom baseURL needed—the provider handles it internally (source: [OpenRouter Vercel AI SDK integration docs](https://openrouter.ai/docs/guides/community/vercel-ai-sdk))

3. **Environment Variable** — Use `OPENROUTER_API_KEY` (not a custom name). Get your free key at [openrouter.ai/keys](https://openrouter.ai/keys) (source: [OpenRouter docs](https://docs.judgmentlabs.ai/documentation/integrations/model-providers/openrouter))

4. **Latest Vercel AI SDK Version** — Version 6.x is current (announced at [vercel.com/blog/ai-sdk-6](https://vercel.com/blog/ai-sdk-6)). Import from `'ai'` package. For structured data, use `generateObject()` instead of `streamText()` (source: [AI SDK Core docs](https://sdk.vercel.ai/docs/reference/ai-sdk-core/generate-object))

5. **Structured Output Pattern** — Use `generateObject()` with a Zod schema. Pass `schema: z.object({ verdict: z.enum(['pass', 'fail']), reason: z.string() })` to get type-safe structured output. The SDK validates the response automatically (source: [AI SDK structured data guide](https://sdk.vercel.ai/docs/ai-sdk-core/generating-structured-data), [Node.js example](https://sdk.vercel.ai/examples/node/generating-structured-data/generate-object))

6. **Best Judge Models on OpenRouter** — For evaluation/judging tasks, top recommendations are:
   - **Google Gemini 3 Flash** (`google/gemini-3-flash-preview`) — 1.05M context, lowest latency, ranked #1 for academia/finance/health evaluations
   - **Claude Opus 4.6** (`anthropic/claude-opus-4.6`) — strongest for long-context reasoning and judgment tasks
   - **DeepSeek V3.2** (`deepseek/deepseek-v3.2`) — cost-efficient with strong reasoning
   - **MiniMax M2.5** (`minimax/minimax-m2.5`) — fastest inference, recent SOTA (Feb 2026)
   (source: [OpenRouter model rankings](https://openrouter.ai/rankings), [Programming collection](https://openrouter.ai/collections/programming))

### Recommended Approach

**Installation & Config:**
```bash
npm install @openrouter/ai-sdk-provider ai zod
```

**Environment:**
```bash
export OPENROUTER_API_KEY="sk-or-v1-your-key-here"
```

**Structured Output Call for Pass/Fail Verdict:**
```typescript
import { createOpenRouter } from '@openrouter/ai-sdk-provider';
import { generateObject } from 'ai';
import { z } from 'zod';

const openrouter = createOpenRouter({
  apiKey: process.env.OPENROUTER_API_KEY,
});

const result = await generateObject({
  model: openrouter('google/gemini-3-flash-preview'),
  schema: z.object({
    verdict: z.enum(['pass', 'fail']).describe('Whether the output passes evaluation'),
    reason: z.string().describe('Concise explanation of the verdict'),
  }),
  prompt: 'Evaluate if this code is production-ready: [code here]',
});

console.log(result.object); // { verdict: "pass", reason: "Code includes error handling and type safety..." }
```

**Notes:**
- OpenRouter normalizes all requests/responses across providers—no baseURL customization needed
- The provider supports `providerOptions` for per-call configuration (routing preferences, etc.) via the unstable feature
- All models on OpenRouter support structured outputs via JSON schema enforcement
- Gemini 3 Flash has the best latency for synchronous judge calls; use Claude Opus for reasoning-heavy evaluations
