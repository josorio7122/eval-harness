## Research: Vite + React + TypeScript Setup with Tailwind v4, shadcn/ui, and Tremor

### Answer

**Tailwind CSS v4** has completely changed its setup for Vite—no more `tailwind.config.js` required. It uses a Vite plugin with a single CSS import. **shadcn/ui v4** (released March 2026) is now fully compatible with Tailwind v4 and includes new presets and skills features. **Tremor** requires Tailwind CSS v4.0+ and is compatible but its Vite guide is marked "Updating Soon"—only Next.js has full documentation.

### Key Findings

#### 1. **Tailwind CSS v4 in Vite** — CSS-first, plugin-based

**Install command:**
```bash
pnpm add -D tailwindcss @tailwindcss/vite
```

**Config files:**
- **NO** `tailwind.config.js` needed (removed in v4)
- **NO** PostCSS config needed (Vite plugin replaces it)
- Single import in `src/index.css`:
  ```css
  @import "tailwindcss";
  ```

**vite.config.ts:**
```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

**Known issues:**
- Many users mistake v3 workflows for v4 and run `npx tailwindcss init -p`, which fails silently
- The old three-line `@tailwind` directives (`@tailwind base`, `components`, `utilities`) are replaced with single `@import "tailwindcss"`
- For advanced customization, use the `@theme` CSS directive instead of JavaScript config
- (Source: https://tailwindcss.com/docs/installation/using-vite, https://devtoolbox.dedyn.io/blog/tailwind-css-v4-complete-guide, updated Feb 2026)

---

#### 2. **shadcn/ui v4** — Full Tailwind v4 + React 19 support

**Init command (standard):**
```bash
pnpm dlx shadcn@latest init
```

**For Vite + React + Tailwind v4 specifically:**
```bash
pnpm dlx shadcn@latest init
# Select Vite template when prompted
```

**New in shadcn/cli v4 (March 2026):**
- **Presets** — Scaffold projects with `--preset` flag: `pnpm dlx shadcn@latest init --preset a1Dg5eFl`
- **Skills** — New `shadcn/skills` gives coding agents context for working with components: `npx skills add shadcn/ui`
- **Dry-run/Diff** — Inspect changes before applying: `pnpm dlx shadcn@latest add button --dry-run`
- **Templates** — Vite now included: `pnpm dlx shadcn@latest init` offers Next.js, Vite, TanStack Start, React Router, Astro, Laravel
- **Monorepo support** — `pnpm dlx shadcn@latest init -t next --monorepo`

**TypeScript path setup (required for Vite):**
Update both `tsconfig.json` and `tsconfig.app.json`:
```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

**Tailwind v4 compatibility:**
- ✅ Full support. All components are updated for Tailwind v4 and React 19
- Components automatically use v4 when initialized (no manual migration needed for new projects)
- HSL colors converted to OKLCH (Tailwind v4 standard)
- `data-slot` attributes added to all primitives
- Deprecated: `toast` component (use `sonner` instead), `default` style (use `new-york`)
- (Source: https://ui.shadcn.com/docs/tailwind-v4, https://ui.shadcn.com/docs/changelog/2026-03-cli-v4)

#### 3. **Tremor** — Requires Tailwind v4.0+ but Vite guide incomplete

**Install command:**
```bash
pnpm add @tremor/react
```

**Additional dependencies (from official docs):**
```bash
pnpm add @headlessui/react @tailwindcss/forms @remixicon/react
pnpm add -D tailwindcss@^4  # Ensure v4+
```

**Tailwind configuration:**
For v4, include Tremor in content paths:
```typescript
import type { Config } from 'tailwindcss'

const config: Config = {
  content: [
    './src/**/*.tsx',
    './node_modules/@tremor/**/*.{js,ts,jsx,tsx}',
  ],
  theme: {
    extend: {
      // Tremor customizations here if needed
    },
  },
}

export default config
```

**Compatibility status:**
- ✅ Tremor requires Tailwind CSS v4.0+ (mandatory)
- ✅ React 18.2.0+ required
- ⚠️ **Vite guide marked "Updating Soon"** — Only Next.js has full v4 setup docs
- Official docs at https://tremor.so/docs/getting-started/installation show Next.js + Tailwind v3.4 examples, but mention v4.0+ requirement
- The npm.tremor.so docs are outdated; the main tremor.so docs are current but Vite section incomplete
- (Source: https://npm.tremor.so/docs/getting-started/installation, https://tremor.so/docs/getting-started/installation, https://www.tremor.so/docs/getting-started/installation)

### Recommended Approach

**For a new Vite + React + TypeScript project with Tailwind v4, shadcn/ui, and Tremor:**

1. **Create the Vite project:**
   ```bash
   pnpm create vite@latest my-app --template react-ts
   cd my-app
   ```

2. **Install Tailwind v4 with Vite plugin:**
   ```bash
   pnpm add -D tailwindcss @tailwindcss/vite
   ```

3. **Update `src/index.css`:**
   ```css
   @import "tailwindcss";
   ```

4. **Update `vite.config.ts`:**
   ```typescript
   import { defineConfig } from 'vite'
   import react from '@vitejs/plugin-react'
   import tailwindcss from '@tailwindcss/vite'

   export default defineConfig({
     plugins: [react(), tailwindcss()],
   })
   ```

5. **Configure TypeScript paths for shadcn/ui:**
   - Update `tsconfig.json` and `tsconfig.app.json` with baseUrl and paths (see above)

6. **Initialize shadcn/ui:**
   ```bash
   pnpm dlx shadcn@latest init
   # Choose Vite when prompted
   # This creates components.json and sets up component directory
   ```

7. **Install Tremor (optional):**
   ```bash
   pnpm add @tremor/react @headlessui/react @tailwindcss/forms @remixicon/react
   ```

**Compatibility notes:**
- Tremor's Vite guide is incomplete but the package works with Tailwind v4—follow the Next.js setup pattern
- shadcn/cli v4 (March 2026) now has full Vite template support
- No `tailwind.config.js` is needed unless you're using the CSS `@theme` directive for advanced customization
- Restart the dev server after config changes
