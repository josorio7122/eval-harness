## Research: Aggregate Statistics in Eval Platform UIs

### Answer

Leading eval platforms (Braintrust, LangSmith, Promptfoo) display **summary statistics at multiple levels**: experiment-wide aggregates (pass rates, mean scores, cost), per-grader/scorer summaries in column headers, and per-example aggregates showing how many graders passed. Stats are shown in summary sidebars, column headers with statistical indicators, and dedicated chart views (histograms, scatter plots). For pass/fail systems specifically, **pass rate** is the primary stat, often shown as a percentage with a count of passing tests.

### Key Findings

#### 1. **Braintrust — Multi-Level Aggregate Display**

**Experiment-level summary** (source: https://www.braintrust.dev/docs/evaluate/interpret-results):
- Visible in **Details panel** (sidebar) showing:
  - Scorers used in the evaluation
  - Datasets tested
  - Metadata like model and parameters
  - Comparisons to other experiments
  - **Experiment summary** pane displays overall metrics

**Score/metric aggregates in column headers**:
- Column headers show **summary statistics** for each score/metric
- Option to **"Order by regressions"** — sorts by regression of a score vs. baseline
- Within **grouped tables**: shows one experiment's summary by default, with option to **"Include comparisons in group"** to show all experiment data

**Per-row aggregates**:
- **Group results** feature: Select `Display > Group by` to group table by metadata fields
- Shows patterns across test cases
- Can compare across multiple experiments within grouped view

**Dedicated summary layouts**:
- `Display > Layout` offers **Summary** view: "Large-type summary of scores and metrics across all experiments"
- Also available: **Grid** (side-by-side outputs) and **List** (default table)

**Aggregate score formulas**:
- Braintrust supports **"aggregate scores"** — formulas combining multiple scores into one metric
- Useful when tracking many scores but need a single overall quality metric

---

#### 2. **LangSmith — Column-Based Aggregates with Comparison Highlighting**

**Per-experiment default metrics** (source: https://docs.langchain.com/langsmith/analyze-an-experiment):
- Default columns show: **input, output, reference output, feedback scores, latency, token counts, cost, status**
- Customizable **column precision**: control decimal places (up to 6) on numeric feedback scores
- **Heat Map threshold** configurable for numeric feedback scores (high, middle, low) to color-code red/green

**Regression/improvement highlighting**:
- Comparison view highlights:
  - **Red** = regression relative to baseline
  - **Green** = improvement
- Column header shows **count of regressions and improvements** for each feedback key
- Click regressions/improvements buttons to filter rows by that metric

**Experiment comparison metrics** (source: https://docs.langchain.com/langsmith/fetch-perf-metrics-experiment):
- Fetch via `read_project()` API returns:
  - `latency_p50`, `latency_p99` (percentile latencies)
  - `total_tokens`, `prompt_tokens`, `completion_tokens`
  - `total_cost`, `prompt_cost`, `completion_cost`
  - **`feedback_stats`**: per-scorer stats including `n` (count), `avg` (mean), **`values`** (categorical counts like `CORRECT: 6, INCORRECT: 3`)
  - `error_rate`
  - `first_token_p50`, `first_token_p99` (for streaming)

**No dedicated summary view** — stats appear inline in columns and accessible programmatically via API.

#### 3. **Promptfoo — Charts & Pass Rate Prominently Featured**

**Results Charts** (source: https://www.promptfoo.dev/docs/usage/web-ui):

1. **Pass Rate**
   - **"Percentage of tests where all assertions passed"**
   - Primary aggregate metric displayed as chart
   - Works for pass/fail systems perfectly

2. **Score Distribution**
   - Histogram of scores per prompt
   - Each test score = **mean of its assertion scores** (see weighted assertions)
   - Shows distribution across test cases

3. **Scatter Plot**
   - Compare two prompts head-to-head
   - Color-coded: Green = Prompt 2 higher, Red = Prompt 1 higher, Gray = same

**Table display**:
- **Display mode toolbar**: Filter by All, Failures, Passes, Errors, Different, Highlights
- **Toolbar filters**: By metrics, metadata, pass/fail with operators (`=`, `contains`, `>`, `<`)
- **Inference details** can be toggled: Tokens, latency, cost, tokens/sec
- **Cell-level scoring**: Mark as passed (1.0), failed (0.0), or custom score (0-1)

**Export options**:
- CSV / JSON for external analysis
- Failed tests only for debugging
- DPO JSON for preference training
- Human Eval YAML for labeling workflows

#### 5. **Metrics Useful for Pass/Fail Eval Systems**

All three platforms support pass/fail evaluation. Most useful metrics:

1. **Pass Rate** (count and %)
   - Promptfoo: Explicit chart
   - Braintrust: In summary/group views
   - LangSmith: Derived from `feedback_stats` values

2. **Pass Count per Grader/Scorer**
   - Braintrust: Grouped by scorer in summary
   - LangSmith: `feedback_stats` includes `{PASS: N, FAIL: M}` counts
   - Promptfoo: Cell marking (👍 pass, 👎 fail)

3. **Failure Count & Failure Rate**
   - Braintrust: Complement of pass rate
   - LangSmith: `error_rate` + scorer-specific failure counts
   - Promptfoo: Display filter for "Failures" view

4. **Per-Item Aggregate** (how many graders passed on a single test case)
   - Braintrust: Row-level in grouped view
   - LangSmith: Row-by-row in table
   - Promptfoo: Cell-level scoring with custom scores

5. **Regression/Improvement Deltas**
   - Braintrust: Column header stats with regression ordering
   - LangSmith: Green/red highlighting + count bubbles
   - Promptfoo: Diff filter mode

### Recommended Approach

**For a pass/fail eval platform UI:**

1. **Experiment Summary Pane** (top-right or sidebar)
   - **Pass Rate**: `X% (N/total)` — large, bold
   - **Failure Rate**: `Y% (M/total)`
   - **Per-grader breakdown**: Table showing each grader's pass count and rate
   - **Comparison to baseline** (if applicable): delta pass rate (+/-X%)

2. **Column Headers** (for per-grader columns)
   - Show **pass count**: `(7/10 passed)` or `70%`
   - Sortable by pass rate for quick scanning

3. **Results Table**
   - **Display filters**: Pass, Fail, All
   - **Per-row indicator**: Badge showing `[2/3 graders passed]` or similar
   - Allow per-cell pass/fail override (toggle button)

4. **Dedicated Chart Section**
   - **Pass rate over time** (if experiments are time-series)
   - **Pass rate per grader** (grouped bar chart)
   - **Failure distribution** (which graders failed most often)

5. **Export/Download**
   - CSV/JSON with `grader_id`, `test_id`, `pass/fail`, `timestamp`
   - Aggregates: total pass rate, per-grader stats
